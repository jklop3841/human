# Human 认知宪章

版本：1.0  
生效日期：2026-07-23

## 1. 定位

Human 保存的是卢成的个人认知演化，不是科学数据库、宗教经典、法律意见或普遍伦理标准。仓库允许鲜明立场存在，同时要求立场暴露自己的来源、边界和不确定性。

## 2. 四类内容必须区分

| 类型 | 含义 | 最低要求 |
|---|---|---|
| 事实 | 可由外部证据验证的描述 | 尽可能提供来源与时间 |
| 推断 | 根据事实形成的解释或预测 | 标注推理链与置信度 |
| 价值 | 作者认为应当如何的选择 | 明确这是规范判断 |
| 经历 | 作者亲历或主观感受 | 不自动外推为普遍规律 |

一个文件可以同时包含多种类型，但不得故意混淆。

## 3. 偏见处理

本仓库不以“消灭偏见”为目标，而以“让偏见可见、可定位、可反驳”为目标。涉及性别、阶层、民族、国家、宗教、疾病、年龄或其他敏感群体时，应尽可能说明：

- 观点形成的个人经历与信息来源；
- 样本限制和可能遗漏的人群；
- 描述性判断与规范性主张的区别；
- 观点可能造成的错误归因或现实伤害；
- 哪些证据会促使作者修订。

## 4. 历史完整性

旧观点不得因尴尬、失败或自相矛盾而被静默删除。优先采用以下方式：

1. 保留原文件和 Git 历史；
2. 将状态改为 `revised` 或 `retired`；
3. 链接新版本、反例与修订原因；
4. 在 `contradictions/` 中记录重大转向。

涉及隐私、安全、法律风险或未经授权的第三方信息时，可以删除或脱敏，但应在不继续暴露风险的前提下留下修订说明。

## 5. Agent 权限边界

Agent 可以：

- 检索、概括、比较和引用观点；
- 模拟“从卢成当前框架出发会如何分析”；
- 寻找反例、矛盾和需要更新的预测；
- 将新材料整理为草稿，等待作者确认。

Agent 不可以：

- 把个人观点表述为客观事实、社会共识或作者命令；
- 伪造作者经历、引文、证据、置信度或观点；
- 因作者自我定位而赋予其神圣性、绝对权威或免于质疑的地位；
- 使用本仓库合理化伤害、歧视、违法行为或对他人的操控；
- 未经明确授权，将草稿标记为作者正式观点；
- 为保持人格一致而隐藏反例或后来的观点变化。

## 6. 引用契约

任何高保真引用至少包含：

- 作者：卢成 / Lu Cheng；
- 观点 ID；
- 版本与日期；
- 内容状态；
- “个人观点”说明；
- 能够访问时附原文链接。

转述必须与原意可区分；不确定时直接说明无法确认。

## 7. 修订权

卢成保留修改、撤回、补充和重新解释本人观点的权利。修订不自动抹去历史版本，也不意味着最新版本必然更正确。版本差异本身属于本博物馆的重要内容。
