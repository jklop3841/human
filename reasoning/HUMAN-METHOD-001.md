# HUMAN-METHOD-001：现实结构编译协议

## 目的

把混乱现实中的信号编译成可质疑、可验证、可执行、可复盘的结构模型和行动资产。

总流程：

`现实信号 → 结构模型 → 竞争解释 → 验真动作 → 决策阈值 → 执行资产 → 监控记录`

## 九步判断协议

### 1. 定锚

写清：

- 谁在做决定；
- 真正要回答的问题；
- 想得到什么结果；
- 时间窗口；
- 可承受损失；
- 错判、漏判、延迟和不行动的代价。

### 2. 建模

列出主体、关系、利益、资源、权限、依赖、叙事、反馈和约束。每条信息标记为：

- 已观察事实；
- 他人转述；
- 推断；
- 比喻；
- 未知。

### 3. 推演

画出主意图线，至少推演四种路径：

- 顺利；
- 受阻；
- 转向；
- 失败。

对每一步写明必要条件、领先信号与主要依赖。

### 4. 证伪

为核心判断写出能推翻它的观察，并至少建立一个竞争模型。不能只收集支持原判断的材料。

### 5. 溯源

追踪关键信息的来源、时效、采样范围、利益关系和可能失真。把“没有数据”与“数据表明没有”分开。

### 6. 评级

分别评价：

- 证据：已观察 / 强推断 / 弱推断 / 未知；
- 因果链：直接 / 间接 / 仅相关；
- 风险：低 / 中 / 高 / 不可逆；
- 决策：立即行动 / 小试 / 等待证据 / 停止。

证据等级的操作定义：

- 已观察：有可直接检查的记录，并注明来源、样本和时间窗；
- 强推断：多个独立信号一致，且因果路径明确；
- 弱推断：只有单一间接信号，或因果路径尚未验证；
- 未知：现有证据不能区分解释。

### 7. 验真

设计能区分竞争模型的最小测试。测试应尽量：

- 便宜；
- 可逆；
- 单一主要变量；
- 有明确时间窗；
- 预先设定成功、失败与停止阈值。

没有基线时，阈值必须标为“暂定”：先收集基线，或由决策人根据可承受损失确认。不得把临时阈值伪装成行业标准。

### 8. 固化

把通过验证的结构转成：

- 清单与 SOP；
- 数据库字段；
- Agent 指令；
- 预测记录；
- 驱动力与负例条目；
- 冗余、恢复和熔断规则。

保留模型版本和适用边界。

### 9. 监控

设定领先指标、复查频率、转向阈值和退出条件。结果出现后回填记录，不允许只保存成功案例。

## 现实范畴迁移编译法

当陌生领域缺少直接理解时，从熟悉范畴中抽取关系结构，再迁移到目标领域。

### 操作步骤

1. 记录目标领域中已经观察到的现实信号。
2. 选择真正熟悉、拥有大量正反案例的源范畴。
3. 抽取源范畴的对象、关系、奖励、惩罚、权限和反馈。
4. 映射到目标范畴，只保留关系结构相似的部分。
5. 主动写出断裂处，不强迫一一对应。
6. 推导目标领域中的可观察预测。
7. 通过外部证据、访谈或小样本测试验证。
8. 将通过验证的结构输出为内容、商业动作、Agent 指令或 SOP。
9. 把正例与负例分别入库。

### 最小迁移表

| 源范畴对象 | 目标范畴对象 | 保持的关系 | 断裂处 | 可验证预测 |
|---|---|---|---|---|
| 谁/什么 | 谁/什么 | 依赖、奖惩、权限或反馈 | 条件差异 | 将观察到什么 |

表面相似不是迁移。迁移必须产生此前没有的、能够被现实否定的预测。

## 结构反演协议

当只看见结果、看不见机制时：

`可见信号 → 候选隐藏对象 → 关系模型 → 竞争假设 → 现实测试 → 更新`

约束：

- 一个信号至少考虑两个解释；
- 不从结果直接推断人的意图；
- 不把暗信息当事实；
- 优先测试会产生不同结果的假设；
- 若测试无法区分模型，继续收集信息或暂停决策；
- 失败后允许模型死亡。

## 人、AI、Agent 与现实的分工

| 角色 | 主要责任 |
|---|---|
| 人 | 观察、直觉、价值边界、可承受损失、小测试与最终选择 |
| AI | 结构化、映射、反演、生成竞争解释、反例和证据计划 |
| Agent | 文件、自动化、数据库、检查项、测试和监控 |
| 现实 | 通过付款、阻力、行动、失败与时间提供最终反馈 |

AI 与 Agent 不得替作者发明未表达的价值判断，也不得把推断写成现实事实。

## 四类长期资产

### 驱动力库

字段：名称、方向、强度、半衰期、适用状态、成功案例、失败案例。

### 预测记录库

字段：输入、数据范围、模型版本、预测、时间窗、判断阈值、实际结果、修订。

### 负例库

字段：错误方向、过早或过晚的转折点、遗漏暗信息、突发冲击、误判反例/干扰、造成的损失。

### 冗余与熔断层

字段：关键依赖、单点故障、替代工具、备用路径、备份、权限边界、恢复时间、止损与退出条件。

## 失败后的强制复盘

每次失败必须归入以下一种：

1. 新的结果干扰类型及其识别信号；
2. 新的暗信息假设及其验证方式；
3. 一条被证伪或需要收缩边界的规则。
4. 证据不足，暂时仍为未知。

缺失、陈旧或失真的测量可以登记为“诊断干扰”，但不能自动解释业务结果。

如果失败没有改变模型、行动或监控方式，复盘就还没有完成。
