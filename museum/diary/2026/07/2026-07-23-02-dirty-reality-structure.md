# 第二章：脏现实——我怎样理解一个真正运行的世界

> 这篇日记由 AI 根据卢成此前持续表达、并要求长期保存的理论整理而成。它不是逐字录音，而是作者观点的结构化版本。正式定义见 `HUMAN-REALITY-001`，操作协议见 `HUMAN-METHOD-001`。

我一直觉得，人很容易把现实想得太干净。

写一份方案，好像只要方案正确，事情就应该照着发生；做一个产品，好像产品足够好，用户就应该购买；讲一个道理，好像证据充足，对方就应该接受。可我见到的现实从来不是这样。现实里有人，有钱，有权限，有时间差，有身体，有平台，有关系，有不愿意说出来的利益，也有我们根本看不见的信息。

所以我把它叫作“脏现实”。

这个“脏”不是骂现实，也不是说人坏。它是说，真正的世界不是实验室里被清理过的变量。一个事情里往往同时站着很多人，每个人都带着自己的目标、恐惧和资源。信息不会完整到达，命令不会自动变成执行，规则写在纸上也不等于谁都有权限使用。中间还会突然撞进政策、平台变化、现金流、疾病、战争、工具故障，甚至只是某个人今天不想配合。

## 现实的底层不是观点，而是关系

我目前最核心的判断是：

`关系 → 结构 → 场 → 路径 → 结果`

一个人拥有什么观点很重要，但他与谁交换、依赖谁、受谁控制、相信谁、能调动什么资源，往往更直接地决定他能走哪条路。关系在一段时间里稳定下来，就形成结构。结构不是一张静止的组织图，它会形成一个场：有些动作顺势，有些动作很费力，有些门看起来存在，但你根本没有钥匙。

最后出现的结果，不只是“某个人做了什么”的结果，而是这个人带着自己的意图进入结构以后，与各种力量碰撞出来的结果。

## 有些东西会把路径压弯

钱、权力、信任、叙事、平台、技术工具、身体、时间、现金流、法律身份、组织资源、社会关系和执行权限，都不是背景。它们会改变一个选择的成本、风险和可行性。

我把这些东西暂时叫作“现实质量体”。这个说法不是物理学，而是一种帮助思考的比喻：有质量的东西会把路径压弯。一个有流量的平台、一段高信任关系、一个能签字的人、六个月现金流，都可能让同一个方案走向完全不同的结果。

如果不能说清楚它具体改变了哪项成本、权限或者概率，那“质量”就只是一句漂亮话。

## 暗信息存在，但不能拿它编故事

现实中一定存在我们不知道的变量。我把那些尚未被看到、却可能改变关系结构的变量叫作暗信息，把尚未被确认的关系叫作暗关系。

暗信息不是阴谋论的许可证。不能因为结果与预期不同，就立刻说背后一定有人操纵。正确的用法应该是：先看预期路径和实际路径在哪里发生偏差，再提出几个候选解释，然后去寻找它们各自会留下的信号。

如果一个解释永远无法被证明是错的，那它就不能帮助决策，只是在保护讲故事的人。

## 一条主意图线，会被很多东西撞击

人做事时，脑子里通常有一条主意图线：我做 A，就会经过 B，最后得到 C。但现实更像一台三维弹球机。政策、平台、现金流、工具、合作关系、身体和偶然事件，会把这条线撞向别处。

有些撞击是真干扰，有些结果则是在证明原来的判断就是错的。这两者必须分开。

如果模型声称某些条件下事情应该成功，而条件确实存在，结果仍然失败，那更接近反例。只有当某个干扰有独立证据、有明确因果路径，最好还在行动前被登记过，才可以把失败暂时归为干扰。否则就应该老老实实写“未知”。

不能每次失败都说：“我的理论没错，只是现实干扰了它。”这样一来，理论就再也不会错，也再也没有用了。

## 现实中的系统会腐烂

一个组织、一段关系、一个流程、一套知识，不会因为建立过一次就永远有效。没有持续投入，它会衰退；没有信息纠错，误差会累积；没有反馈，执行会漂移；没有边界维护，责任和权限会混乱；没有更新，曾经有效的路径会过期。

我借用“热力学”和“熵增”来描述这种现象，也只是比喻。真正有用的是把它落到现实指标上：多久没人维护了，反馈延迟多长，错误积压多少，数据是否陈旧，现金还能撑多久，关键成员是否正在离开。

## 人会进入一个事件世界

一个人被改变，不只是因为听到一句话。他会进入一个由叙事、权威、群体反馈、时间压力、早期奖励、沉没成本和未知信息组成的事件世界。

一开始的一点奖励，会让他继续投入；继续投入会增加沉没成本；周围人的反馈会让某种解释显得越来越真实；最后，他不是在孤立地判断一个观点，而是在一整套结构里行动。

这既可以解释创业、组织、消费和传播，也提醒我们注意风险：一个事件世界可能形成错误的吸引子，使人越来越难退出。因此，分析一个系统时不能只问它如何把人吸进去，还要问它有没有退出路径、反向证据和熔断条件。

## 我怎样理解陌生的东西

我经常先从自己熟悉的现实范畴出发。

但这不是简单地说“商业像战争”“传播像宗教”。表面像没有意义。真正应该迁移的是关系结构：里面有哪些对象，谁奖励谁，谁惩罚谁，谁掌握权限，什么反馈会让路径自我加强。

比如我把推理小说迁移到说服写作，迁移的不是侦探和凶手，而是“读者通过线索自己完成结论”这条关系。只要关系还能保持，它就可能帮助我理解陌生领域；一旦对应关系断裂，就必须停下来验证。

我把这叫作“现实范畴迁移编译法”，也可以叫“熟悉范畴反推法”。

## 最终必须变成行动

一套现实理论如果只能解释过去，就还不够。它必须能被编译成下一步行动。

我的基本协议是：

`定锚 → 建模 → 推演 → 证伪 → 溯源 → 评级 → 验真 → 固化 → 监控`

先说清楚到底要判断什么；再画出主体和关系；推演不同路径；主动找可能推翻自己的证据；追溯信息来源；给证据和判断评级；用小实验验真；把有效东西固化成规则、SOP 或 Agent 文件；最后继续监控现实有没有变化。

人在其中提供观察、直觉、价值边界和愿意承担的损失。AI 帮助结构化、生成竞争解释、寻找反例和验证办法。Agent 把它变成文件、自动化、数据库、检查项。现实则通过付款、阻力、行动、失败和时间，给出最终裁决。

以后每次失败，都不应该只剩下一句“没做成”。至少要沉淀为三种东西中的一种：一条新的干扰线、一种待验证的暗信息，或者一条已经被证明无效的规则。

这就是我目前的现实解释操作系统。它不是标准答案，更不是科学定律。它是一个人面对复杂世界时，试图不被干净模型欺骗，也不被自己的故事欺骗的办法。
